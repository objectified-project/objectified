Why this exists

The platform already generates a complete OpenAPI 3.1 document per version. A one-click hosted mock server turns that spec into a live, schema-valid API — letting frontend and partner teams build against a contract before any backend exists. It closes the design → consume loop with near-zero added backend (responses are derived from existing schema examples). Ranked Phase 1 in FUTURE_OFFERINGS.md; previously no mockup existed.

What's new

One-Click Provision

Design notes

  • Mocks are generated from the existing GET /v1/swagger/{tenant}/{project}/{version} output — no separate schema source to maintain.
  • Response bodies prefer schema example/examples, falling back to faker generation constrained by the primitive type and format.
  • Free-tier mocks auto-expire (e.g. 24h) and are rate-limited; paid tiers are persistent with custom domains. Reuses the Monetization metering pipeline.
