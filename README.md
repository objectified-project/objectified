# Objectified

Objectified is an OpenAPI 3.1.0 Specification Application that provides a visual editor
for creating and editing Schema Objects and Properties.

## Goals

The Objectified application is a work in progress.

The goals of the project are:

- Provide user and group tenancy for schema definitions and sharing
- Provide visual editing of schemas class and property definitions
- Provide a database for storing schemas
- Provide a visual editor to create REST schemas using OpenAPI 3.1.0 Specifications

Eventually, the project will provide a database for storing data according to the
defined schemas.

## The Story

This is the 4th iteration of the project, effectively started in 2001 with Webplasm (see Webplasm
database, now defunct.)

Its inception for what the application would become effectively started around the time I was
working as a contractor back in 2012 for a large hurricane modeling firm.

Webplasm was the first of its kind to develop an object database, which is where this application
originally got its start.  The idea was to create dynamic database table designs using a series
of MySQL table objects: an object, property, object-property, object instance, and property-value
table with indices and links, all programmatically handled.

What this application evolved into is a visual editor to shape your data, along with storing instances
of that data (eventually).  Data can be dynamically shaped and reshaped as needed, stored in both
PostgreSQL and MongoDB databases for consistency.
