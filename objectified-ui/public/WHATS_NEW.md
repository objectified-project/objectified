## Repositories
- Added REPO-10.5 **saved scan-report filters**: persist up to 20 named presets per user under `user_settings.repository.scan_report.saved_filters`, tenant APIs under **`GET/POST/PATCH/DELETE …/dashboard/{tenant}/repository_scan_report_saved_filters`**, **`SAVED_FILTER_LIMIT_REACHED`** when capped, single **default** preset applied on **Scanned repository report** landings with no query string, and ADE **Saved filters** / **Save current filter** plus **Manage saved filters** on `/ade/dashboard/repositories/reports`.
- Added REPO-12.6 **auto-import audit ↔ change-report linkage**: each `repository.auto_imported` workflow-audit row carries `changeReportId`, counts, and summary rollup in **`detail`**, and **`GET …/workflow-audit?action=repository.auto_imported`** projects **`changeReport`** (`id`, **`summaryKind`**, **`breakingChangeCount`**) for governance queries without reverse joins.
- Added REPO-12.5 **repository import in-app notifications** (`odb.repository_import_notification`): emit once per `(tenant, import job, recipient)` when auto/manual import needs attention, targeting the user who last toggled **Import** (`importSelectionActorId`), respecting `user_settings.notifications.repository` toggles (`auto_import_failed`, `auto_import_breaking`, `auto_import_warnings`, `mapping_required`), deep-link payloads for Sync / Specs, **Mark as reviewed** clears matching rows, GET `/v1/dashboard/{tenant}/repository_import_notifications` + unread count, and a 15-minute Enterprise email digest gate (SMTP stubbed).
- Added REPO-12.4 **persisted `repository_scan_report` artifact** (PostgreSQL): one row per `(repository_id, scan_id)` with Epic E `totals_json`, bounded `payload_json` (≤5k file rows) plus optional overflow blob URL, `attention_score`, tenant retention override (Enterprise), daily purge keeping the latest report per repository, and `repository.scan_report.purged` workflow audits; finalize shares a transaction with terminal `repository_scan` updates via `Database.finalize_repository_scan_with_report`.
- Added REPO-12.3 **version provenance on committed auto-import**: validated `versions.repository_source` tuples are written with each committed `commit-sha` auto-import push, `repository_file.last_imported_checksum` / `last_imported_version_id` update in the same transaction, `repository.auto_imported` audit rows include `repositoryId`, `branch`, `path`, `commitSha`, `projectId`, `versionId`, and `actorId`, and checksum-based skip (`unchanged_checksum`) avoids duplicate revisions by reading prior imports from PostgreSQL.
- Added REPO-11.4 **per-repository Issues** tab (between **Scans** and **Sync history** when `repository_attention` has open reasons): `GET /v1/repositories/{tenant}/{id}/attention` (paths per reason), **Issues** at `/ade/dashboard/repositories/{id}/issues` (rewritten to `?tab=issues`), one card per reason with Fix / Import deep links (linked accounts, manifest, files, sync, scans, specs), 5s client refresh + resolved-card fade, and shared dashboard attention broadcast.
- Added REPO-11.3 **Recent Imports Needing Attention** on `/ade/dashboard` directly under REPO-11.2: one `GET /v1/dashboard/{tenant}/recent_imports_attention` call (repository-scoped import jobs from the last 7 days that failed, need review, have breaking changes, OpenAPI diff warnings, or conflicts), row links to the sync tab change report, **Mark as reviewed** with per-user dismissals in `odb.user_settings`, and **All imports** to the repositories index.
- Added REPO-11.2 **Repositories Needing Attention** on `/ade/dashboard` (right column, above **Recent Activity**): one `GET /v1/dashboard/{tenant}/repository_attention?limit=5` call, top reason chip, file count, age, and deep link to the repository (Specs tab for `stale_checksum`, otherwise the tab best matching the reason set), empty state, **View scan report** to `/ade/dashboard/repositories/reports`, optional healthy footer, 5s refresh and shared `BroadcastChannel` with corpus stats.
- Added REPO-11.1 **repository attention rollup** (backend): `odb.repository_attention` materializes per-repo “needs attention” reasons and a weighted score; the REST service recomputes on scan completion, import jobs, spec selection changes, auto-pause, and an hourly safety pass, and issues `pg_notify` on `dashboard.attention` when the row is stored in PostgreSQL (for REPO-11.3 and future UI).
- Added REPO-10.4 **scan report export** at `/ade/dashboard/repositories/reports`: `POST /v1/repositories/{tenant}/scan-reports:export` (async job, CSV or NDJSON line stream with `file` + per-row metadata) with a 100k-row cap, `GET …/exports` and job polling, cancel/retry, and tokenized download; ADE **Export** menu, **Recent exports** list, toasts, and a Next `…/content` proxy that uses the session tenant.
- Added REPO-10.3 **corpus stats strip** at `/ade/dashboard/repositories/reports` (clickable roll-up of repositories tracked, importable specs, awaiting selection, parse errors, and manifest errors; each applies the corresponding table status filter). Served by `GET /v1/dashboard/{tenant}/repository_corpus_stats` (tenant-scoped `repository.read` roll-up refreshed on scan report materialization) with a matching `odb.repository_corpus_stats_rollup` table for future DB-backed storage; the UI refreshes the strip via `BroadcastChannel` and tab visibility without blocking the report table.
- Added REPO-10.2 **per-repository report drill-in** at `/ade/dashboard/repositories/{id}/reports/{reportId}` and `/…/reports/latest` (resolves the newest materialized row, then `router.replace` to a stable ID) with `GET /v1/repositories/{tenant}/{id}/scan-reports` (list) / `.../latest` (302) / `.../{reportId}` (detail) / `.../{a}/diff/{b}`; UI summary (totals, attention, scan metadata), read-only files snapshot, collapsible file/scan error panels (REPO-6.3 style), and compare-with-previous totals and path-churn.
- Added REPO-10.1 **Scanned repository report** at `/ade/dashboard/repositories/reports` with server-paginated `GET /v1/repositories/{tenant}/scan-reports` (pre-aggregated per-scan `totalsJson` from in-memory `repository_scan_report` rows, plus `lastReportId`), provider/status/search filters in the query string, default sort by attention then last scan, and row links into REPO-10.2.
- Added REPO-12.1 **auto-import dispatch chain**: scan completion applies selection (`discovered`), mode (`ready_to_promote`), then checksum (`unchanged_checksum`) before queueing `repository_auto_import` jobs; emits `repository.auto_imported` on dispatch, uses per-repository advisory-style serialization, and extends `repository_file_status` with `discovered` / `ready_to_promote`.
- Added REPO-12.2 **project mapping conflicts** so auto-import can park specs in `mapping_required` with typed conflict reasons, emit `repository.spec.mapping_required` / `repository.spec.mapping_resolved` audit rows, add a `mapping_required` attention reason, and ship a Specs-tab / drawer “Map to project” flow (map existing, create in-memory project, or defer) with `PATCH .../specs/{fileId}/mapping` plus a Next API proxy.
- Added REPO-9.7 **bulk spec selection** on the repository Specs tab: sticky selection toolbar, **Select all matching filter** (cursor-paginated, capped at the 500 ID REST limit), **Set auto-import** / **Clear auto-import** (set requires every selected row to already have import opt-in; otherwise the action is disabled), confirmation when bulk-disabling import for **already imported** files, optimistic bulk apply with full rollback and toast on failure, and **chunked** bulk POSTs when a selection exceeds 500 IDs.
- Added REPO-9.6 **spec detail drawer** with a header (file path, branch, format chip, status pill, copy-link, Open in provider), a lazy-loaded read-only Monaco preview (with Show raw toggle, language inferred from format/path) capped at 2 MB before swapping to a download prompt, derived lint summary chips (errors/warnings/info), the last five import jobs with change-report links, mirrored Import / Auto-Import switches, deep-linking via `?fileId=…`, and Esc-to-close. Backed by new REST endpoints `GET /v1/repositories/{tenant_slug}/{repository_id}/specs/{file_id}/detail` and `GET /v1/repositories/{tenant_slug}/{repository_id}/specs/{file_id}/content`.
- Added REPO-9.5 **Import Now** (`POST .../specs/{fileId}:importNow`) to queue a manual dry-run import when `import_enabled` is on (independent of auto-import), with 30s idempotency per file/branch, `repository_manual_import` job `sourceKind`, `repository.spec.import_now_triggered` audit, and Specs tab / drawer actions that link to the Sync history tab; specs list a new `importing` status while a job is `pending_review`.
- Added REPO-9.4 ADE Repository Detail "Specs" tab listing importable specs (confidence ≥ 50%) with format/status/last-import metadata, optimistic Import + Auto-Import toggles with REST 4xx rollback and atomic auto-import cleanup, deep-linkable filter chips (`?status=…`), bulk-edit toolbar, an Import Now overflow action, and a spec detail drawer.
- Added REPO-9.3 repository specs REST endpoints for cursor-paginated listing (`GET /v1/repositories/{tenant_slug}/{repository_id}/specs`), single-spec selection updates, and transactional bulk selection updates with typed invariant errors and `repository.spec.selection_changed` audit rows.
- Added REPO-9.2 per-spec `auto_import_enabled` on `repository_file` (default false), including `PATCH /v1/repositories/{tenant_slug}/{repository_id}/files/{file_id}/auto-import-enabled`, independent `repository.spec.selection_changed` audit discrimination by `field`, scan dispatch gating that requires both import and auto-import enabled, and automatic clearing of auto-import whenever import is disabled.
- Added REPO-9.1 per-spec `import_enabled` on `repository_file` (default off without manifest; manifest-listed specs use explicit `importEnabled` with legacy-safe omissions), selection preserved across re-scans, `PATCH /v1/repositories/{tenant_slug}/{repository_id}/files/{file_id}/import-enabled` plus `repository.spec.selection_changed` audit, and import-job dispatch only when the flag is true.

## Studio AI
- Added the Studio AI chatbot launcher and panel: a floating bubble in the bottom right of the canvas opens a slide-out panel with full-screen mode, toggled with `Cmd+Shift+A` / `Ctrl+Shift+A`.
- Filled out the Studio AI chat surface to the design guidelines: distinct user/assistant bubbles, typing indicator, markdown rendering, syntax-highlighted code blocks with a copy button, regenerate / thumbs up / thumbs down message actions, and a one-click import button when an assistant reply contains an OpenAPI spec in a ```json``` block.
- The Studio AI chatbot is now context-aware: each message you send carries a snapshot of your current project, version, classes, reusable properties, and canvas selection, and a "Sharing context" chip in the panel lets you inspect exactly what the assistant can see.
- The Studio AI chatbot now handles multi-turn conversations: follow-up prompts like "add a phone field", "remove priceCents", "make name required", "rename id to productId", "make it more like Stripe Charges", or simple clarification questions are recognized as edits to the previous reply, and the refined OpenAPI spec is re-shipped so you can keep iterating without restarting the thread.
- The Studio AI chatbot now persists conversations per project and version: a new toolbar above the chat surfaces "New", "History", "Export", and "Clear" actions, the history view lets you browse and search every saved thread, exporting downloads the active conversation as a markdown file, and the chat automatically restores your most recent thread when you reopen the panel.
