## Git Behavior
- **Delta pull (#2592):** **GET** **`/v1/versions/{tenant}/{project}/{revisionId}`** and **`.../by-version/{versionLine}`** accept **`sinceRevisionId`** — response includes **`schemaPullDelta`** ( **`removedSchemaNames`**, full **`schemas`** entries for touched entities at head, formal **`guarantee`** string). Unknown id / wrong project / non-ancestor **since** → **400** with **`UNKNOWN_REVISION_ID`**, **`REVISION_PROJECT_MISMATCH`**, or **`SINCE_NOT_ANCESTOR_OF_HEAD`**
- **Compatibility report UI (#2590):** **ADE → Versions** merge and rollback dialogs show compatibility findings **grouped by severity and entity path** with **rule hits**; when **`compatGateOnMerge`** blocks merge, **tenant admins** can **override** with a **required justification** recorded in **`workflow_audit`** as **`version.compat_gate_override`** (**REST** **`compatGateOverrideReason`** with **`skipCompatGate`**); the **Commit** dialog explains **post-commit** **`schema.compatibility`** audit logging
- **Compatibility engine (#2589):** **objectified-rest** exposes a **`CompatibilityCheckEngine`** (deterministic **`overall`**, findings, **`ruleHits`**) used by **`POST .../compatibility`** and recorded after successful **push** (parent→head) and **merge** (target tip→merged revision) in **`workflow_audit`** with **`action`** **`schema.compatibility`**
- **Webhook delivery retry & dead-letter (#2588):** **objectified-rest** persists **delivery attempts** per event, applies **bounded backoff** (10s / 60s / 300s, up to four tries), marks terminal **dead-letter** failures, and exposes **GET** **`.../push-webhook-subscriptions/{tenant}/deliveries/dead-letter`** and **`.../deliveries/{eventId}`**; optional **Fernet** key encrypts signing secrets at rest for outbound **HMAC** delivery
- **Push webhook subscriptions (#2587):** **objectified-rest** exposes tenant-scoped **CRUD** for HTTPS **push webhook** endpoints — **signingSecret** is write-only; list/get/patch responses return **signingSecretRef** only (secret stored hashed); duplicate normalized URLs return **409** **`WEBHOOK_URL_DUPLICATE`**
- **Published immutability (#2586):** Published revisions default to **immutable** for git-like writes — **push**, **merge**, and **rollback** return **409** **`PUBLISHED_IMMUTABLE`** unless a **tenant administrator** retries with **`overridePublishedImmutability`** and **`overrideReason`** (recorded in **`workflow_audit`** as **`version.immutability_override`**); **`publishedImmutable`** on publish sets the flag; **read** APIs unchanged
- **Studio draft lock chip (#2585):** **Studio** header shows a live **draft lock** chip (owner, countdown, absolute UTC in tooltip) when a lock is active on the selected draft revision; **`GET`** **`.../draft-lock`** plus **10s** polling
- **Draft lock API (#2584):** Unpublished revisions support **edit locks** — **`POST`** **`.../draft-lock/acquire`**, **`renew`**, **`release`**, tenant-admin **`force-release`**; second editor gets **409** **`DRAFT_LOCK_CONFLICT`** with **`ownerUserId`** and **`expiresAt`**; run DB migration **`20260412-150000.sql`** for **`odb.version_draft_lock`**
- **Branch protection / merge path (#2583):** Branches may require **merge workflow** instead of **direct push** — column **`require_merge_path`** or project **`metadata.branchPushPolicy.patterns`** (e.g. `main`, `release/*`); blocked pushes return **403** with **`code: MERGE_PATH_REQUIRED`**; **tenant admins** may bypass; **`PATCH`** branch policy (**REST** and **Studio** API) sets **`protected`** / **`requireMergePath`**
- **Rollback audit trace (#2582):** **REST** `POST .../version-branches/rollback` accepts optional **`reason`**; **`workflow_audit`** rows for **`version.rollback`** include **`detail.fromRevision`**, **`detail.toRevision`**, optional **`detail.reason`**, plus **`actorId`** and **`createdAt`** on the row; **`metadata.rollback`** may include **`reason`**; **ADE** rollback proxy forwards **`reason`**
- **Rollback confirm + impact (#2581):** After **Preview impact** on **Rollback branch**, **Apply rollback** opens a Radix **confirmation** with **revision id**, **UTC time**, and **entity-level impact** from the API (`impactSummary`); **Roll back** applies; the timeline refreshes on success
- **Compare with current (#2580):** Each **version row** actions menu includes **Compare with current** — opens the **Compare Version Schemas** dialog with a diff from the **selected revision (base)** to the **latest project head** (newest `created_at` in the loaded list); **light/dark** styling matches the existing diff viewer; closing returns to the **timeline** with **search / author / date** filters unchanged
- **History timeline filters (#2579):** **ADE → Versions** includes a **Timeline** bar — **search** (revision note, changelog, commit body), **author**, **From** / **To** dates, and **Reset**; the **revision graph** and **table** show the filtered set; **REST** list versions supports **`q`**, **`creatorId`**, **`createdAfter`**, **`createdBefore`**
- **Workflow audit ledger (#2577) & audit API (#2578):** **objectified-rest** records **push, pull** (revision GET), **merge**, and **rollback** to **`odb.workflow_audit`**, and exposes **`GET /v1/versions/{tenant}/workflow-audit`** with **filters** (action, actor, outcome, revision, project, date range) and **offset or cursor** pagination (`schemaVersion`, **`hasMore`**, **`total`**); see **`objectified-rest/docs/WORKFLOW_AUDIT_API.md`**; run DB migration **`20260412-120000.sql`** for the table
- **Merge apply gating (#2576):** **Apply merge** requires a **Preview merge** for the current branch pair; **Apply** stays disabled until **every** conflict row has **Mine / Theirs / Manual**; merge **409** responses use **`MERGE_UNRESOLVED_CONFLICTS`** with **`unresolvedCount`** and a clear message
- **Bulk merge conflict resolution (#2575):** In the **Merge branches** conflict panel, filter paths and **conflict type**, then **Bulk (shown)** or **Bulk (all)** **Mine** / **Theirs** to set many rows at once; **per-row** buttons still override after a bulk apply
- **Merge conflict list (#2574):** **Versions** branch merge dialog shows a **conflict table** (path, type from merge-preview **`kinds`**, resolution state) with **Mine** (target), **Theirs** (source), and **Manual** (opens placeholder for future side-by-side submit); unresolved rows stay **highlighted** until you pick a resolution (stored locally until merge-session resolution APIs accept payloads)
- **Merge session persistence (#2573):** Optional **`persistMergeSession: true`** on **`POST .../version-branches/merge-preview`** creates **`merge_sessions`** + per-path **`merge_session_conflicts`** (initial status **`preview`**); **`GET .../merge-sessions/{id}`** and **`GET .../merge-sessions/{id}/conflicts`**, **`PATCH .../merge-sessions/{id}`** for **`resolving` / `applied` / `aborted`** with **`merge_session_status_events`** audit
- **Merge preview API (#2572):** **Dry-run** `POST .../version-branches/merge-preview` returns **`summaryCounts`** (two-way diff), **`conflictCounts`**, structured **`conflicts`** (`path` + `kinds`: threeWay / blend / twoWay), **`classification.addedSchemaNames`**, optional **`mergedOpenApi`** when auto-merge fits the size cap (or omit via **`includeMergedOpenApi: false`**); **`dryRun: true`** — no persistence unless **`persistMergeSession`**
- **Branch from history (#2571):** **ADE → Versions** revision **graph** nodes include **Branch → Branch from here** (same Radix dialog as the table); suggested **`feature/…`** branch name; create uses **`/api/projects/.../version-branches/from-revision`** (REST lineage); success selects the **new branch tip** for commits and scrolls into view
- **Branch from revision (#2570):** **REST** `POST .../version-branches/from-revision` creates a **named branch** whose tip is an existing revision, stores **`branchedFromRevisionId`** lineage on **`version_branches`**, rejects duplicate names per project (**409** `BRANCH_NAME_CONFLICT`), and is **idempotent** when the same name + source tip already match (**`idempotentReplay`**)
- **Sync status chips (#2569):** **Studio** header shows compact **Dirty** (canvas auto-save pending), **↑N** (your revisions on the lineage from latest to the selected revision), and **Ahead** (newer revision than selection or active **STALE_HEAD** conflict) with Radix tooltips
- **Conditional pull (#2568):** **GET** revision (`/v1/versions/{tenant}/{project}/{revisionId}` and **by-version**) returns **`ETag`** (revision id); **`If-None-Match`** with the same id yields **`304`** with an empty body; **`/api/versions/[versionId]`** forwards conditional headers for Studio pulls
- **Push 409 banner (#2567):** When a commit **push** returns **`409`** (**`STALE_HEAD`**), **Studio** and **Versions** show a **persistent banner** with **Pull** (switch to server head / refresh list) and **Open merge** (dashboard merge flow, **`?merge=1&projectId=`** deep link); banner clears after **pull** or **successful merge**
- **Push optimistic lock (#2566):** **REST** create revision (`POST /v1/versions/...`) requires **`baseRevisionId`** (and **`branchName`** when the project has multiple branches); omitted **`baseRevisionId`** → **`422`**; provided base that does not match the current head → **`409`** with **`STALE_HEAD`** and current head metadata; empty-project bootstrap requires an empty base and otherwise returns **`400`** with **`INVALID_BASE`**; ADE **Commit** sends the branch tip as **`baseRevisionId`**
- **Pre-commit policy (#2565):** **REST** enforces **revision note** rules and **max JSON payload** size before persisting commits (create/fork/update/publish/merge/rollback); optional per-project **`metadata.commitPolicy`** (`requireShortMessage`, `maxCommitPayloadBytes`, char caps); **`400`** responses use **`POLICY_VIOLATION`** or **`PAYLOAD_TOO_LARGE`** with a message (global default payload cap via **`COMMIT_POLICY_MAX_PAYLOAD_BYTES_DEFAULT`**)
- **Commit metadata (#2563):** Schema revisions persist optional **`author`**, **`message`**, and **`externalRef`** (REST create/fork + revision list/detail); stored as `commit_author`, `commit_message`, `external_ref` — existing rows surface **null** safely
- **Commit dialog (#2564):** ADE **Versions** **Commit** opens a Radix dialog with required **Message**, optional **External reference** (max 500 chars), inline validation, disabled submit until valid, and **sonner** toasts; new revisions post to **REST** via `/api/versions` with commit metadata

## Versioning
- **Successor redirect (#749):** **GET** a schema revision with **`successorResolution=resolve`** (JSON body is the replacement revision; **`X-Objectified-Resolved-From`** / **`X-Objectified-Successor-Chain`** headers) or **`redirect`** (**HTTP 307**); optional **`auditSuccessorResolution`** for audit rows. **ADE** `/api/versions/[id]` forwards those query params and response headers; **`redirect`** is re-mapped to the portal URL
- **Sunset dates (#748):** Schema revision metadata supports **`sunsetAt`** (UTC ISO), optional **`deprecatedAt`** for ordering, and **`successorRevisionId`** (required when a sunset is set). **Edit Version** picks local **datetime** and sends UTC to the API; tenant admins can update deprecation metadata on **published** revisions. Sunset timeline entries expose **`sunsetAt`** alongside **`sunsetDate`**
- **Migration guide (#747):** **Compare versions** includes a **Migration guide** tab — ordered steps for **breaking** changes between the selected revisions, revision-pair metadata and template version, links to the **Breaking doc** (#746) and compatibility (#506); **Copy**, **Append to compare-to changelog**, **Download Markdown**, or **Download PDF**; optional **Author notes** from `breaking:` lines in revision changelogs (#502)
- **Breaking changes doc (#746):** **Compare versions** includes a **Breaking doc** tab — auto-generated Markdown from the schema diff (stable `components.schemas…` ids; **Breaking** / **Additions** / **Other**), **Copy** or **Append to compare-to changelog** to store with revision notes
- **Version rollback (#745):** **Revert-style** rollback — **REST** `rollback-preview` / `rollback` create a **new** revision whose classes match a **prior ancestor** of the branch tip (`parent_version_id` = old tip; `metadata.rollback` records lineage); **compatibility** tip→target (#506), optional **`compatGateOnRollback`**, **`version.rollback`** audit; **ADE → Versions** row menu **Rollback branch to this revision…** with preview and confirmation
- **Revision history graph (#744):** **ADE → Versions** graph uses **left-to-right lanes** (git-style), **violet merge commits** vs **emerald branch tips** with **branch name** labels and hover; **branch chips** filter history by **ancestor union** so merges stay readable; same interactions as **#743** (compare, spec, **Load older**)
- **Revision history graph (#743):** **ADE → Versions** shows an interactive **DAG** of revisions (primary and merge parent links) with zoom/pan — **click** a node to compare it to its **primary parent**, **Ctrl/Cmd-click** to view the OpenAPI spec; merge revisions show **two incoming edges** (dashed = merge parent); **Load older** widens the newest-first window for large projects
- **Canvas compare (#742):** **ADE → Versions → Compare** includes a **Canvas** tab — **split** or **overlay** views of saved Studio layouts (loaded when you open the tab), with a **legend** for added / removed / moved / unchanged nodes and edges vs the logical OpenAPI tabs
- **Property-level diff (#741):** Expandable class rows show **readable property lines** (e.g. `type` / `default` / `nullable` with **old → new**), broader OpenAPI field coverage on compare, **cap + show all** for very large classes, and **Merge branches** preview lists **conflict paths grouped by class** (same `schemas.*` IDs as Schema Changes)
- **Compare classes (#740):** **ADE → Versions → Compare** → **Schema Changes** tab lists **every** `components.schemas` class with **+/−/~** (added / removed / modified / unchanged), **search**, **virtualized** scrolling for large APIs, expandable **property-level** lines per class, and **Copy class stat** (plain-text export)
- **Lifecycle tags (#739):** Revisions support governance labels **Stable / Beta / Deprecated / Archived** (`metadata.lifecycle`); **Versions** list has a **lifecycle filter** and badges; **deprecated** aligns with **#507**. **Archived** revisions are read-only except tenant admins (lifecycle / revision lock). Distinct from **git-like release tags** (#501)
- **Sunset timeline (#508):** **REST** `GET /v1/versions/{tenant}/sunset-timeline` aggregates **deprecated** and **sunset** revisions across projects (optional **`projectId`** filter); **ADE → Sunset timeline** lists **timeline** status (announced / imminent within 30 days / past), **lifecycle** (deprecated vs sunset reached), **#507** warning text, and **CSV export**
- **Deprecation warnings (#507):** **Revision-level** deprecation in **`versions.metadata`** (`deprecated`, `deprecationMessage`, `successorRevisionId`, `sunsetDate`); **compatibility** responses include **`deprecationWarnings`** (with migration guide **#747**), optional **`409`** via **`policy.http409WhenDeprecatedRevision`**, and project **`failCiOnDeprecatedRevision`** for CI-style **blocked** flags; exported OpenAPI includes **`info.x-objectified-revision-deprecation`**; **Studio** and **Migration** headers show amber **banners** when the selected revision(s) are deprecated
- **Backward compatibility (#506):** **REST** `POST /v1/versions/{tenant}/{project}/compatibility` compares two **revision IDs** (generated OpenAPI) and returns **safe / breaking / unknown** findings, deterministic **reportFingerprint**, link to **#746** when breaking, optional **409** via `policy.http409WhenBreaking`; **ADE → Versions → Merge branches** runs the check after merge preview (target tip vs source tip). Optional project metadata **`compatGateOnMerge`** sets **tenantCompatGateActive** and blocks apply when the pair is not fully safe
- **Create version from copy (#505):** **ADE → Versions → Create new version** shows **branch context** when copying from a revision—**branch tip picker** when multiple named branches exist, **breadcrumb + compact ASCII lineage** from parent links, **merge** called out in text (not color-only), and **screen-reader summary**; branch list refresh and loading/error states when metadata fails to load
- **Protected versions (#504):** **Tenant-admin policy** on **tags**, **named branches**, and **revisions** — `protected` / `revision_locked` columns, **403/409**-style errors from API routes, **version_protection_audit** for denials and policy changes; **ADE → Versions** shows **Protect / Unprotect**, **Lock revision**, and shields on locked rows
- **Fork versions (#503):** **Fork to another project** from the version row menu — copies the source revision’s schema into a **different** project with stored lineage (`forkedFromRevisionId`, optional upstream project for merge-back); **REST** `POST /v1/versions/{tenant}/{projectId}/fork`; **not** the same as a **named branch** in the same project (#500)
- **Revision notes & changelogs (#502):** **REST** returns `shortMessage` / `changelog` (aliases for `description` / `change_log`) with **tenant-default size limits**; **publish** applies last-minute notes in one transaction; **ADE → Versions** uses **Revision note** / **Changelog** labels, validation before create/edit/publish, **compare** shows both sides’ notes and **breaking:** bullets for downstream docs
- **Version tags (#501):** **Git-like tags** on schema revisions — create/list/delete from **ADE → Versions** (optional message, channel, **immutable** lock); **REST** `GET/POST /v1/version-tags/{tenant}/{project}` and `PATCH/DELETE .../{tag_id}`; compare dialog can **set base/compare from a tag**; history table filters by tag
- **Versions (#500):** **Named branches** (tip = version snapshot), **merge preview** and **apply merge** with optimistic `baseRevisionId` / target tip; merge revisions record **two parent** version ids; conflict banner when overlapping schema paths differ (merge-base LCA planned in #2593)
