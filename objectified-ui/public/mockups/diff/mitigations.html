User.age · required field removed

CRITICAL · 5 affected consumers · -18 score pts

BREAKING

Deprecate with sunset window

AI

Mark User.age as deprecated: true for v3.0 and v3.1, then remove in v4.0. Consumers see a deprecation warning header during the window.

Impact reduction: -85% · Low complexity

Versioned endpoint

Keep /v2/users serving the legacy schema (with age) and route v3+ clients to /v3/users. Old consumers continue to work indefinitely.

Impact reduction: -100% · High complexity

User.phone · maxLength 20 → 15 + deprecated

HIGH · 4 affected consumers · -6 score pts

BREAKING

Field aliasing — keep phone + serve contacts[]

AI

Serialize phone alongside contacts[type=phone].value for one major version. Existing code keeps working; new code can adopt the richer model.

Impact reduction: -90% · Medium complexity

Loosen the constraint

Restore maxLength: 20 in v3.0; keep deprecation in place. Avoids any payload-validation breakage during the deprecation window.

Impact reduction: -100% (validation) · Low complexity

User → Order · cascade nullify → delete

HIGH · 2 affected consumers · -8 score pts

BREAKING

Consumer-specific feature flag

Gate the cascade-delete behavior behind a per-consumer flag. Existing analytics-etl keeps nullify; new consumers opt-in to delete.

Impact reduction: -75% · Medium complexity

Default value injection

Default new cascade to nullify at the storage layer; only callers that explicitly set delete see the new behavior.

Impact reduction: -95% · Low complexity
