Migration guide

acme.platform/User · v2.1.0 → v3.0.0 · 11 steps · 4 breaking · ~12.5h estimated

Open tracker
Progress:
2 / 11
Breaking changes 4 actions · ~9h
DONE

1. Stop sending User.age in payloads

Removed in v3.0.0. Backend will return 400 if present. Endpoint: POST /v3/users · /properties/age

Before · v2.1.0
const user = {
  name: "Alice",
  email: "a@x.io",
  age: 32,
  phone: "+1-555-0100",
};
After · v3.0.0
const user = {
  name: "Alice",
  email: "a@x.io",
  dateOfBirth: "1992-03-14",
  phone: "+1-555-0100",
};
DONE

2. Replace flat phone with contacts[]

Field is deprecated and removal targeted for v4.0.0. Migrate to contacts[] with typed entries.

Before
user.phone = "+1-555-0100";
After
user.contacts = [
  { type: "phone", value: "+1-555-0100" },
];
TODO

3. Update User → Order delete handlers

Cascade now delete instead of nullify. Audit any code that depended on orphaned Orders surviving a User delete.

Action
// If you previously did this:
const user = await api.users.delete(id);
const orphans = await api.orders.list({ userId: id });
//                                ^^^^^^^^ will return [] in v3

// Use a soft-delete instead, or fetch orders before deleting:
const orders = await api.orders.list({ userId: id });
await archive(orders);
await api.users.delete(id);
TODO

4. Tighten Address.zipCode validation client-side

Pattern is now ^\d{5}(-\d{4})?$. Matches US ZIP and ZIP+4. International addresses must use the new Address.region reference.

Non-breaking adoption 5 actions · ~3h

5. Adopt new User.dateOfBirth

Optional · ISO 8601 date · gives age computation a stable basis. Age field on legacy responses is no longer maintained.

6. Pass User.locale for localization

Optional · BCP-47 string. Drives email templates and admin UI defaults.

7. Handle new moderator role enum value

Enum widened from [admin, user] to [admin, user, moderator]. Make sure switch statements have a default branch.
