API Paths mockups

Visual designs for the Objectified API Paths Designer — the OpenAPI 3.1 surface shipped at /ade/studio/paths where teams describe their HTTP API as a graph of paths (URL templates) and operations (GET · POST · PUT · PATCH · DELETE) wired to request/response schemas drawn from the same project the Data Designer manages. Mockups cover the canvas, single-path and single-operation editors, the reusable libraries (parameters, responses, security schemes, servers) and a code view that round-trips OpenAPI YAML / JSON. The proposal also expands what the Paths Designer could become: a one-click CRUD generator that derives REST endpoints from any class, contract-test runs, deprecation timelines, and policy-checked publish flows.

"Paths" in Objectified means OpenAPI URL paths (e.g. /customers/{id}). Distinct from JSONPath and from /mockups/mdm/lineage.html which traces per-field provenance through golden records.
