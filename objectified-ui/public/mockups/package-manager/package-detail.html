Stripe Payment Schemas

Production-tested JSON Schemas for Stripe's core payment objects. Use these to validate webhook payloads, generate types, or share a single source of truth across your services.

Installation

obj install @stripe/payment-schemas

What's included

  • charge.schema.json — Charge object with capture, refund, dispute, and metadata fields
  • customer.schema.json — Customer with default source, addresses, tax IDs
  • refund.schema.json — Refund with reason, status, and balance transaction

Usage

Import a schema and validate any payload using your favourite JSON Schema validator (Ajv, jsonschema, etc.):

import charge from '@stripe/payment-schemas/charge';
import Ajv from 'ajv';

const ajv = new Ajv({ strict: false });
const validate = ajv.compile(charge);

if (!validate(payload)) {
  console.error(validate.errors);
}

Compatibility

Schemas track the latest Stripe API version. Major version bumps follow Stripe's API versioning (2024-04-10 → 2025-01-09).

Contributing

Issues and PRs welcome at github.com/stripe/payment-schemas. Run obj test to validate locally before publishing.
