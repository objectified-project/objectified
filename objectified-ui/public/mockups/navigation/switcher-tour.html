Tour overview

One shell, four very different apps.

Each stop below is a faithful snapshot of how an existing app from /mockups would render under the two-tier header. The Tier 1 row stays globally identical — logo, app pill, context slot, search, notifications, help, plan-usage pill, avatar — but its individual elements morph (the app pill icon and label, what the context slot shows, sometimes even the right-side utility pill) to match the active app. The Tier 2 row is owned by the app entirely.

What this proves: the navigation isn't an abstraction that only works for the Designer canvas. It's a contract that adapts to every shape the suite needs — from split-pane IDEs to REST API editors, GitHub-style repo views, and analytics dashboards.

What stays vs what swaps

  • Stays · Logo, search, notifications, help, plan-usage pill, avatar, app-pill shape
  • Swaps · App-pill icon & label, context slot contents, accent color of active sub-nav item
  • Owned by app · The entire Tier 2 sub-nav strip and its grouping

Three things this tour proves.

If the shell can absorb four shapes this different — IDE workspace, REST editor, repo manager, analytics dashboard — it can absorb the rest of the suite without bespoke rework.

The shell is a contract, not a layout.

Tier 1 declares slots: app-pill, context, search, utilities, plan/utility-pill, avatar. Each app fills its own context slot and chooses what utility-pill to show. The shell stays predictable; the app stays in control.

Sidebar nav becomes Tier 2.

Every app in the suite had its own ~280 px sidebar. Folding those into a horizontal Tier 2 with subtle group separators reclaims a screen-wide canvas for every app and removes the question "which sidebar level am I in?".

Tool panels stay; navigation goes.

Designer keeps its class list and inspector. Paths keeps its outline. Git keeps the branches table layout. The new shell only replaces what was navigation — not panels that exist to do work.

Each frame is a static snapshot. In a live build, switching apps via the app pill or ⌘+K swaps Tier 2 + context slot in place — no full page reload.
