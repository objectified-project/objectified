Step 1 of 4 · Breaking · User.phone

Migrate User.phone to the new constraints

The phone field is now deprecated and its maxLength has tightened from 20 to 15 characters. New clients should use the contacts[] array, where each entry carries an explicit type (phone, email, fax). Existing payloads with phone numbers longer than 15 characters will be rejected by the v3.0.0 validators.

Why this changed: Internationalisation requirements forced contacts to be a typed list. The 15-character cap aligns with E.164 maximums.

Transformation helper stripped non-digits, capped to 15 chars 
function migratePhone(legacy) {
  const digits = legacy.replace(/[^\d+]/g, '').slice(0, 15);
  return [{ type: 'phone', value: digits }];
}
