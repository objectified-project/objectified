Discriminated unions in OpenAPI 3.1

Use a discriminator property to make oneOf schemas self-describing — and to keep your client codegen happy.

A discriminated union (sometimes called a tagged union) is a oneOf schema where one property — the discriminator — tells consumers which variant they are looking at. This is crucial for code generators, runtime parsers, and humans reading your API responses.

Tip

Most generators only emit useful typed code when a discriminator is present. Without it you get generic envelopes.

payment.yaml · OpenAPI 3.1 
Payment:
  oneOf:
    - $ref: '#/components/schemas/CardPayment'
    - $ref: '#/components/schemas/BankTransfer'
    - $ref: '#/components/schemas/Wallet'
  discriminator:
    propertyName: type
    mapping:
      card:     '#/components/schemas/CardPayment'
      transfer: '#/components/schemas/BankTransfer'
      wallet:   '#/components/schemas/Wallet'
