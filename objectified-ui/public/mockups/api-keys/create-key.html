Choose key type

Select how this key will be used. Different types carry different scope models and connection mechanisms.

Browser Access

For REST API, web and mobile clients. Authenticates via Authorization: Bearer or query parameter. Supports read / write / admin scopes and IP allowlisting.

  • REST & GraphQL APIs
  • Web / mobile / server-to-server
  • IP allowlist & rotation policy
  • Prefix: orcl_live_ / orcl_test_

MCP Server

For AI agent frameworks and IDE extensions connecting via the Model Context Protocol. Grants access to MCP tools with fine-grained per-tool permissions.

  • Cursor, Claude Desktop, VS Code
  • Streamable HTTP or stdio transport
  • Tool-level permission scopes
  • Prefix: odb_mcp_

The full secret is shown once immediately after issuance. It is stored as a bcrypt hash — if lost, you must rotate to generate a new secret.

Security checklist

  • Secret stored as SHA-256 → bcrypt (raw value never persisted)
  • Scopes follow principle of least privilege
  • Rotation policy applies (90-day max age)
  • IP restriction not configured (MCP Server keys are tenant-scoped by default)
