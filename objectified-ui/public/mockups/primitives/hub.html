Primitives

Mockup suite · 4 screens

Proposed UX improvements for managing reusable JSON Schema primitive definitions — used as building blocks when designing classes and properties across a tenant's projects.

Key design improvements

Split-pane editor — form builder on left, live JSON schema preview on right (instead of a stacked modal)
Visual category picker — icon cards for string / number / integer / boolean / array / object
Expandable rows — click a primitive to expand its schema inline in the table
Import wizard — step-by-step: Source → Select definitions → Review & Done
System primitive lock — clear visual affordance, edit/delete disabled, hover tooltip explains why
Live schema validation — AJV status badge in preview pane (Valid / Invalid)

Colour system

Violet — primary, create flow Indigo — string category, header accents Sky/Blue — system (read-only) primitives Emerald — tenant primitives, valid schema Amber — number/integer category, warnings Rose — delete, invalid schema, danger Purple — array category Teal — object category Pink — boolean category
