Merge Key Configuration

Define how incoming rows are matched to existing objectified instances per source table

UPSERT

Match on merge key; update if exists, insert if new. Most common strategy for OLTP sources.

INSERT ONLY

Append new rows; never update existing. Suitable for immutable event tables.

SOFT DELETE

Set a deleted_at timestamp on rows missing from source. Preserves history.

HARD DELETE

Remove rows from objectified-db that are absent from source. Destructive — use with caution.
