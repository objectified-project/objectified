Repository Store mockups

Visual designs for browsing source repositories that belong to a user's Linked Accounts (GitHub, GitLab, Bitbucket) or to public Git URLs. Files are catalogued in Postgres, then filtered with comma-separated globs, regex, or a preset picker for known importable formats (OpenAPI, Arazzo, AsyncAPI, JSON Schema, GraphQL, Protobuf, & more). Selecting a file fetches its bytes on demand, classifies it, and lets the user map it to an existing project — or seed a new project from spec metadata — and run the existing importer to produce a new project version.

Persistence: Postgres only. No in-memory stores. See README.md for full notes intended for the LLM that implements this feature.
