Developer Mode Concept Preview

Programmatic, code-first interface to the Designer & Paths canvases — without leaving the web app.

Code first, canvas always

Developer Mode brings VS Code-grade authoring directly into the Objectified web app. Power users can edit schemas and paths as .objschema.json, .objpath.yaml, or via the typed @objectified/sdk — and watch the Designer or Paths canvas update live. Same project, same versions, same lint rules — no IDE install required.

How this fits the existing app

  • 1. Toggle Settings → Developer Mode (or ⌘⇧D). The Designer and Paths header gain a new Code tab next to Canvas.
  • 2. The same document model backs both views. Saving in Code is identical to saving from the Designer — same versioning, branches, PR review, lint, and AI suggestions.
  • 3. The LSP runs in a Web Worker (compiled to WASM from the same crate as the VS Code extension), so IntelliSense, diagnostics, and quick-fixes are byte-identical.
  • 4. The SDK Playground gives power users a safe, sandboxed scripting surface for bulk operations that would be tedious in the Canvas.
