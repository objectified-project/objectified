Shadow

Log violations, never block consumer responses. Use during rollout to discover non-compliant upstreams safely.

Recommended for first 7 days

Enforce

Return 502 Bad Gateway with problem-details body when upstream response fails the schema contract.

Promote after shadow window is clean

Strip

Quietly remove unexpected fields and forward the sanitized response. Useful when upstream leaks internal fields.

Limited to additionalProperties drift

Rollout playbook

  1. 1

    Enable shadow mode

    Violations are logged with route id, upstream URL, and failed-checks summary. No consumer impact.

  2. 2

    Triage drift in violation log

    Sort by frequency, file upstream-fix tickets, or convert minor drift to strip mode.

  3. 3

    Promote to enforce

    When violation rate is 0 for 48h, click Promote to enforce. Future drift returns 502 with problem-details.

  4. 4

    Wire alerts

    Stream violation events to PagerDuty / Slack via the gateway event bus for proactive incident response.
