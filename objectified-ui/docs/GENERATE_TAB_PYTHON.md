# Generate Tab - Python DTO Generation

## Overview

The Generate tab in the Objectified Studio allows you to automatically generate Data Transfer Objects (DTOs) from your class definitions in the Canvas. The first supported language is Python, which generates dataclasses based on your schema.

## Features

### Language Support
- **Python**: Generates Python dataclasses with proper type hints and decorators
  - More languages coming soon (TypeScript, Java, C#, Go, etc.)

### Generated Output
- Single file output: `schema.py` for Python
- Complete with:
  - Dataclass decorators
  - Type hints (including Optional, list, dict)
  - Nested class definitions
  - Field descriptions as comments
  - Proper imports (datetime, UUID, etc.)
  - Project metadata in docstring header

### User Interface

The Generate tab provides:
1. **Language Selector**: Dropdown to choose the target programming language
2. **Monaco Editor**: Syntax-highlighted code editor showing generated code
3. **Copy to Clipboard**: Quick button to copy all generated code
4. **Export to File**: Download button to save as a file (e.g., `schema.py`)

## How It Works

### Data Flow
1. Classes and properties are defined in the Canvas
2. When switching to the Generate tab, the system:
   - Retrieves all class definitions with properties
   - Converts JSON Schema types to Python types
   - Handles nested objects and arrays
   - Resolves class references
   - Generates properly formatted Python code

### Type Mapping

| JSON Schema Type | Python Type |
|-----------------|-------------|
| string | str |
| string (format: date) | date |
| string (format: date-time) | datetime |
| string (format: uuid) | UUID |
| string (format: email) | str |
| integer | int |
| number | float |
| boolean | bool |
| array | list[T] |
| object | dict[str, Any] or nested class |
| $ref | Referenced class name |

### Naming Conventions
- **Class names**: Preserved as-is from Canvas (no conversion applied)
- **Field names**: Converted to snake_case (e.g., `firstName` → `first_name`)
- **Nested classes**: Generated with PascalCase from property name (e.g., `address` → `Address`)

### Optional Fields
- Fields not in the `required` array are marked as `Optional[Type]`
- Default value is set to `None` for optional fields
- Lists and dicts use `field(default_factory=...)` for mutable defaults

### Nested Objects
- Inline object properties generate nested dataclasses
- Arrays of objects generate item classes (e.g., `OrderItem` for `items` array)
- Nested classes are defined before their parent class

## Usage Example

### Input (Canvas Classes)
```typescript
// Person class with properties
{
  name: "Person",
  properties: [
    { name: "name", type: "string", required: true },
    { name: "email", type: "string", format: "email", required: true },
    { name: "age", type: "integer", required: false },
    { name: "address", type: "object", properties: [
      { name: "street", type: "string" },
      { name: "city", type: "string" },
      { name: "zipCode", type: "string" }
    ]}
  ]
}
```

### Output (Generated Python)
```python
"""
E-Commerce API
Version: 1.0.0

Data Transfer Objects for E-Commerce API

Generated by Objectified Studio
"""

from dataclasses import dataclass, field
from typing import Optional, Any

@dataclass
class Address:
    """Home address"""
    street: str  # Street address
    city: str  # City name
    zip_code: Optional[str] = None  # ZIP/Postal code

@dataclass
class Person:
    """A person with basic information"""
    name: str  # Full name of the person
    email: str  # Email address
    age: Optional[int] = None  # Age in years
    address: Optional[Address] = None  # Home address
```

## Implementation Details

### Files Modified
- `src/app/ade/studio/page.tsx`: Added Generate tab UI and state management
- `src/app/utils/python-dto.ts`: Python DTO generator utility

### Key Functions
- `generatePythonDTOs(classes, options)`: Main generator function
- `mapTypeToPython(propData)`: Maps JSON Schema types to Python types
- `generateClass(classData)`: Generates a single dataclass
- `generateField(prop, allProperties, isRequired)`: Generates a field with type hints
- `generateNestedClass(prop, allProperties, className)`: Generates nested classes

### State Management
- `generatedCode`: Stores the generated Python code
- `generateLanguage`: Currently selected language (default: 'python')
- `generateCopied`: Tracks copy button state

## Notes

- The generator preserves the structure and relationships defined in the Canvas
- All field descriptions are included as inline comments
- The code is formatted for readability with proper indentation
- Empty classes generate with `pass` statement
- References between classes are preserved using class names
